# Lab 11.9.3: VLSM Design and Implementation

## 1. Mô tả bài Lab
- **Mục tiêu:** Thiết kế và phân hoạch mạng con sử dụng kỹ thuật VLSM (Variable Length Subnet Mask) để tối ưu hóa không gian địa chỉ IP.
- **Yêu cầu:** Chia dải mạng mẹ `172.16.6.0/24` thành 3 subnet cho 3 phòng ban với số lượng host khác nhau.

## 2. Bảng phân hoạch IP (IP Allocation Table)
| Phòng ban | Nhu cầu Host | Prefix | Subnet Mask | Network Address |
| :--- | :--- | :--- | :--- | :--- |
| **Room 114** | 27 | /27 | 255.255.255.224 | 172.16.6.0 |
| **Room 279** | 25 | /27 | 255.255.255.224 | 172.16.6.32 |
| **Room 312** | 12 | /28 | 255.255.255.240 | 172.16.6.64 |

## 3. Cấu hình tiêu biểu (Command Reference)

### Cấu hình cổng Router (Gateway)
```ios
! Cấu hình cho Room 114 (Ví dụ)
interface GigabitEthernet 0/0
 ip address 172.16.6.1 255.255.255.224
 no shutdown
 exit

```

### Cấu hình IP quản trị Switch (VLAN 1)

*Lưu ý: Switch Layer 2 cần IP để quản trị từ xa.*

```ios
interface vlan 1
 ip address 172.16.6.66 255.255.255.240
 no shutdown
 exit
! Lệnh cực kỳ quan trọng để ra internet:
ip default-gateway 172.16.6.65 

```

## 4. Các điểm cần chú ý (Troubleshooting Checklist)

* **Lỗi Gateway trên Switch:** Nếu không gõ `ip default-gateway`, bạn sẽ không thể Telnet/SSH vào switch từ một mạng khác.
* **Lỗi Default Gateway trên PC:** Nếu PC không cấu hình đúng địa chỉ Gateway của phòng, nó sẽ không thể gửi dữ liệu ra ngoài phân vùng mạng hiện tại.
* **Subnet Mask:** Phải kiểm tra kỹ bảng phân hoạch. Gõ nhầm `224` thành `240` (hoặc ngược lại) là toàn bộ mạng sẽ "tê liệt" ngay lập tức.
* **Thứ tự chia mạng:** Luôn ưu tiên chia cho phòng có nhu cầu nhiều Host nhất trước để tránh lãng phí IP.

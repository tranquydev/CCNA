## 1. Mục tiêu bài Lab
- Cấu hình cơ bản cho Switch (S1, S2).
- Thiết lập địa chỉ IP cho Management Interface (VLAN 1).
- Kiểm tra kết nối mạng bằng lệnh Ping.

## 2. Bảng địa chỉ (Addressing Table)

| **S2** | VLAN 1 | 192.168.1.254 | 255.255.255.0 |
| **PC1** | NIC | 192.168.1.1 | 255.255.255.0 |

## 3. Câu lệnh cấu hình (Commands)
*Dưới đây là các lệnh đã thực hiện trên cả S1 và S2:*

```ios
! Cấu hình cơ bản
hostname S1               ! (Thay bằng S2 cho switch thứ 2)
enable password class
line console 0
password cisco
login
exit

! Cấu hình Banner
banner motd "Authorized access only. Violators will be prosecuted to the full extent of the law."

! Cấu hình IP Management
interface vlan 1
ip address 192.168.1.253 255.255.255.0  ! (IP S2 là 192.168.1.254)
no shutdown
exit

! Lưu cấu hình vào NVRAM
copy running-config startup-config

```

## 4. Ghi chú & Giải đáp lý thuyết (Dùng để ôn thi)

* **Câu hỏi: Tại sao cần `no shutdown` trên VLAN 1?**
* *Đáp trả:* Mặc định cổng ảo (SVI) đang ở trạng thái `down`. Cần lệnh này để kích hoạt giao diện hoạt động.


* **Câu hỏi: Tại sao Switch cần IP nếu nó đã hoạt động ở Layer 2?**
* *Đáp trả:* IP trên Switch (gán cho VLAN 1) không dùng để chuyển mạch dữ liệu, mà dùng để **quản trị thiết bị từ xa** qua giao thức Telnet/SSH.


* **Câu hỏi: Tại sao Ping lần đầu thường bị mất gói (timeout)?**
* *Đáp trả:* Do tiến trình **ARP**. Thiết bị cần gửi gói tin quảng bá (Broadcast) để hỏi địa chỉ MAC của đích trước khi gửi dữ liệu ICMP.
